# Классификация игровых комнат (tiers)

## Назначение

Комнаты делятся на **четыре класса** — бронзовые, серебряные, золотые и платиновые — по **входному депозиту (`entry_fee`)**. Класс определяет ожидаемый уровень ставок, визуальное оформление в лобби и шаблоны демо-комнат при сидировании БД.

## Матрица классов

| Класс     | Диапазон депозита (бонусы) | Аудитория              | Акцент в UI   |
|-----------|----------------------------|------------------------|---------------|
| **Бронза**   | 100–499                    | Новички, низкий риск   | тёплая бронза |
| **Серебро**  | 500–1499                   | Обычные VIP            | нейтральное серебро |
| **Золото**   | 1500–3499                  | Высокий чек            | золото Stoloto |
| **Платина**  | 3500+                      | Топ VIP                | платина / кристалл |

## Правила расчёта

1. Поле `tier` в ответах API вычисляется функцией `resolve_room_tier(entry_fee)` в `app/room_tiers.py`.
2. При создании комнаты можно передать `tier` явно; иначе он берётся из `entry_fee`.
3. Фронтенд показывает бейдж класса в карточке лобби и в шапке комнаты.
4. Класс влияет на шаблоны сидирования и стили бейджей (CSS-классы `participant-chip--bronze` и т.д.).

## Шаблоны по умолчанию (демо)

| Класс    | entry_fee | max_players | boost_cost | boost_mult |
|----------|-----------|-------------|------------|------------|
| Бронза   | 250       | 4           | 50         | 0.10       |
| Серебро  | 1000      | 4           | 150        | 0.15       |
| Золото   | 2500      | 4           | 300        | 0.20       |
| Платина  | 5000      | 4           | 500        | 0.25       |

## Тайминг раунда

- Ожидание набора игроков перед стартом: **15 секунд** (`default_wait_seconds` в `app/config.py`, `WAIT_SECONDS` во фронте).
- Длительность анимации рулетки с кейсами согласована с бэкендом (`opencase_animation_seconds`).

## Никнеймы внутри комнаты

- В списке участников показывается `display_name` — одноразовый псевдоним фиксированной длины 8 символов.
- Псевдоним создаётся на backend при входе участника в комнату и уникален в пределах комнаты.
- После окончания раунда показывается **реальный** ник победителя (`winnerUsername`).

## Заполнение БД

Скрипт `app/seed.py` создаёт таблицы, пользователей и комнаты **всех четырёх классов**, для каждой комнаты добавляет уникальных ботов-участников (без пересечения между комнатами).

Запуск:

```bash
cd /path/to/kazik-game
python -m app.seed
```

## Room Tier Classification (EN)

### Purpose

Room segmentation for Stoloto VIP Opencase: entry threshold, visual tone, and reward intensity.

### Tier Matrix

| Tier | Deposit Range | Target Audience | Visual Accent |
|------|---------------|-----------------|---------------|
| Bronze | 100–499 | New / low-risk players | warm bronze |
| Silver | 500–1499 | Regular VIP users | silver |
| Gold | 1500–3499 | High-value users | gold |
| Platinum | 3500+ | Top-tier VIP users | platinum / crystal |

### Functional Rules

1. Each room exposes a `tier` field in API responses.
2. Tier is derived from `entry_fee` unless explicitly set.
3. Frontend shows tier badges in lobby cards and room header.
