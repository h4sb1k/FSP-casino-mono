# Инцидент: mismatch победителя рулетки

## Кратко

В комнате наблюдался критичный рассинхрон: победитель, выбранный RNG на backend, не совпадал с пользователем, на котором останавливался курсор рулетки на frontend.

## Симптомы

- В `ROUND_RESULT` приходил один `winnerParticipantId`.
- В UI блок победителя и позиция курсора иногда показывали разных участников.
- Пользователь визуально видел "нечестный" результат.

## Причины

1. **Неверная интерпретация `winIndex` на frontend**  
   `winIndex` от backend — это индекс ячейки в ленте, а не индекс участника в массиве игроков.

2. **Отсутствие гарантии winner в позиции `winIndex`**  
   Лента строилась повторением участников, и в ячейке `winIndex` мог оказаться любой элемент.

3. **Перезапись ленты во время раунда (race condition)**  
   После получения `ROUND_RESULT` лента могла перезаписываться fallback-данными из `participants` при последующих обновлениях состояния.

4. **Случайный финальный сдвиг (`randomOffset`)**  
   Даже при корректной ленте визуально создавалось ощущение промаха по карточке.

## Что исправлено

### Frontend: строгая привязка winner к `winIndex`

В `frontend-react/src/features/room/RoomPage.tsx`:

- добавлена генерация ленты раунда, где:
  - элемент на позиции `winIndex` принудительно равен `winnerParticipantId`;
  - остальные элементы заполняются участниками.
- победитель для оверлея берется из того же источника.

### Frontend: защита от перезаписи ленты в раунде

В `frontend-react/src/features/room/RoomPage.tsx`:

- добавлен флаг `roundStripLocked`;
- после `ROUND_RESULT` лента блокируется;
- fallback-обновления ленты отключены в статусах `locked/running/finished`;
- блок снимается на `ROUND_FINISHED`.

### Frontend: предсказуемая остановка курсора

В `frontend-react/src/features/room/components/CaseRoulette.tsx`:

- убран случайный финальный сдвиг: `randomOffset = 0`;
- курсор останавливается точно по центру целевой карточки.

## Важная деталь про RNG

Backend RNG работал корректно: победитель вычислялся заранее и отправлялся в `ROUND_RESULT`.  
Проблема была в визуализации и синхронизации ленты на frontend, а не в самой RNG-логике.

## Как проверить после фикса

1. Собрать React-бандл для backend-страницы:
   - `cd frontend-react && npm run build`
2. Запустить backend:
   - `python3 -m app.main`
3. Открыть `http://localhost:8000` и сделать hard refresh (`Ctrl+Shift+R`).
4. Прогнать 5-10 раундов:
   - пользователь под курсором на `winIndex` всегда совпадает с `winnerParticipantId`.

## Риски на будущее

- Любые изменения генерации ленты должны сохранять инвариант:  
  **`strip[winIndex]` всегда соответствует `winnerParticipantId`.**
- Нельзя перегенерировать боевую ленту после получения `ROUND_RESULT` до завершения раунда.
