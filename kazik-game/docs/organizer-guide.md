# Руководство организатора — Stoloto VIP Opencase

## Обзор

Документ описывает, как настраивать параметры игровых комнат и как эти параметры влияют на игровой процесс и экономику.

## Доступ к конфигуратору

1. Откройте фронтенд: `http://localhost:8000/`
2. Перейдите в раздел «Конфигуратор» (админ-панель)

## Параметры комнаты

### `max_players` — максимальное количество игроков
- **Диапазон**: 2–10
- **Влияние**: определяет размер пула участников и частоту заполнения ботами
- **Рекомендация**: 4–6 для баланса скорости и разнообразия

### `entry_fee` — входной взнос (бонусы)
- **Диапазон**: 100–5000
- **Влияние**: чем выше взнос, тем больше фонд, но меньше доступных игроков
- **Риск**: при `> 5000` и `max_players < 4` → MEDIUM риск

### `prize_pool_pct` — доля призового фонда
- **Диапазон**: 50–95%
- **Влияние**: определявает, какая часть общего фонда идёт на выигрыш
  - Низкое значение (50–60%) → меньше мотивация, больше «сжигания»
  - Высокое значение (85–95%) → привлекательнее, но требует аккуратных бустов
- **Риск**: `> 85%` + дешёвые бусты → HIGH риск

### `boost_enabled` — включить бусты
- **Значения**: `true` / `false`
- **Влияние**: если выключено, участники не могут покупать приоритет

### `boost_cost` — стоимость буста
- **Диапазон**: 50–1000
- **Влияние**: дешёвые бусты при высокой `prize_pool_pct` создают перекос в пользу платящих
- **Риск**: `boost_cost < entry_fee × 0.2` при `prize_pool_pct > 85%` → HIGH риск

### `boost_multiplier` — множитель веса при бусте
- **Диапазон**: 10–50%
- **Влияние**: насколько увеличивается шанс победы при покупке буста
- **Риск**: `> 40%` → HIGH риск (слишком сильное преимущество)

### `bot_win_policy` — политика победы бота
- **`return_pool` (по умолчанию)**: призовой фонд раунда возвращается в общий `promo_pool`
- **`burn`**: фонд «сжигается» (не возвращается, не выводится)

## Система валидации

При изменении параметров система автоматически оценивает риск:

| Уровень | Цвет | Действие |
|---------|------|----------|
| **LOW** | зелёный | Сохранение разрешено |
| **MEDIUM** | жёлтый | Сохранение разрешено с предупреждением |
| **HIGH** | красный | Сохранение **заблокировано** |

### Как читать предупреждения

- **Призовой фонд слишком большой + дешёвые бусты** → система нестабильна, возможны убытки
- **Высокая стоимость входа при малом числе игроков** → низкая заполняемость
- **Слишком высокий множитель буста** → нарушение公平性

## Примеры настройки

### Быстрая комната (low-risk)
```
max_players: 4
entry_fee: 1000
prize_pool_pct: 80%
boost_enabled: true
boost_cost: 200
boost_multiplier: 20%
bot_win_policy: return_pool
```
**Ожидаемый результат**: средняя длительность раунда ~60–90с, умеренный интерес.

### High-roller (medium-risk)
```
max_players: 4
entry_fee: 5000
prize_pool_pct: 85%
boost_enabled: true
boost_cost: 500
boost_multiplier: 25%
bot_win_policy: return_pool
```
**Ожидаемый результат**: крупные призы, но меньше участников, требуется мониторинг заполняемости.

### Агрессивная (high-risk — не сохранится)
```
max_players: 2
entry_fee: 5000
prize_pool_pct: 90%
boost_enabled: true
boost_cost: 100
boost_multiplier: 45%
```
**Результат**: HIGH risk → сохранение заблокировано.

## Мониторинг

После запуска rooms:
- Следите за `total_pool` и `prize_pool` в лобби
- Анализируйте историю раундов (`/api/history`) — частота побед ботов, средний выигрыш
- Отслеживайте `promo_pool_ledger` — возвраты от ботов

## Поддержка

При проблемах:
1. Проверьте логи сервера (stdout)
2. Убедитесь, что `vip_opencase.db` доступна для записи
3. Для сброса конфигурации — удалите запись из `admin_configs` и перезапустите `python -m app.seed`
